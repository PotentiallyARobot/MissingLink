Interpreter (CLI)

The PromptFlow Interpreter is a Python package that runs .promptflow bundles locally, searches the Process Graph, and lets you publish your own work from the terminal.

1. Install (recommended — pip):
pip install promptflow-interpreter
1b. Or download the source archive:
curl -L /api/promptflow/interpreter/latest.tar.gz | tar xz
cd promptflow-interpreter
pip install -e .
2. Authenticate (paste an API key from above):
promptflow login
3. Use it:
promptflow search "youtube channel"
promptflow run shopify-listing --input image=@shoe.png
promptflow init my-process
promptflow whoami

What is PromptFlow™?

A graph economy for AI workflows. Build once. Earn when reused.

The idea

Most AI tools are one-shot generators — you type a prompt, you get an output, the work disappears. PromptFlow turns successful workflows into processes: reusable, named, runnable assets that other people can call directly.

When you describe what you want, the planner picks the right model, recipe, or saved process and runs it. If a saved process already does what you need, it gets used. If not, the planner builds one — and you can save that for next time.

How earning works

Publish a process and set a markup (charged on every run) and a helpful-bonus (held in escrow, released when a runner gives 👍 feedback). When someone runs your process, you earn.

When your process is composed into a bigger one — used as a step inside someone else's workflow — you earn a depth-decayed share of your declared price at every level. The top-level author keeps their full markup; nested authors get a fractional cut at each composition layer (½ per level, capped at depth 6, cycle-safe).

The same cascade applies when your process is adapted. When another user forks your process and refines it via chat, every run of that adaptation pays you a depth-decayed share. Fork lineage and composition lineage combine — if a fork of your process is composed into someone else's workflow, both layers contribute to your share.

Composing or adapting other authors' processes is cheaper, more reliable, and earns more than rebuilding. That's the graph economy.

Trust & quality

Every process accumulates run signals: helpful / unhelpful feedback, failure count, composed-by count. These combine into a per-process quality score and trust score the planner uses to rank candidates. Composing high-trust processes raises your own process's trust — the system rewards reusing what already works.

Forks & learning

When you chat with a published process to refine it, the system creates a private fork for you and learns a tagged preference (e.g. "prefers vertical aspect ratio", "wants under 4 scenes"). Future runs of related processes apply your preferences automatically.

Forks aren't free of obligation — every run of a fork pays the original author a depth-decayed share of their declared price. The forker keeps their adaptation private, the original author keeps earning when their work powers it.

The .promptflow file format

Every process can be downloaded as a .promptflow bundle — a ZIP archive that's also a runnable Python file. Inside: __main__.py, process.py, an editable prompts/ directory, manifest.json, and the original Python listing. Run it with python my-process.promptflow, or unzip and edit prompts as plain text.
